#### 2. Описание логики маппинга

Для обеспечения сопоставимости данных между разными облачными провайдерами была использована следующая классификация:

* **Башня Database:** Включает все сервисы управления данными. Основной упор сделан на семейство `Relational Database` (Azure SQL), что соответствует RDS в AWS.
* **Башня Storage:** Сюда отнесены все ресурсы хранения данных. `Azure Blob Storage` сопоставлен с `Object Storage` (аналог S3), а `Managed Disks` — с `Block Storage` (аналог EBS).
* **Башня Cloud Services:** Использована для PaaS-решений, инструментов разработки и вычислений. Сюда вошли:
* **Compute:** Виртуальные машины (аналог EC2).
* **Developers Tools:** Инструментарий Azure DevOps.
* **Application Services:** Azure App Service (аналог Elastic Beanstalk).
* **Analytics:** Azure Data Factory (аналог Glue/MSK).


* **Башня Network:** Использована для учета трафика (`Bandwidth`), что позволяет прозрачно сравнивать затраты на передачу данных.
