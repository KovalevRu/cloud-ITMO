## Используемые bad practices в плохом CI

### 1. Использование `ubuntu-latest`
**Почему плохо:**  
При обновлении образа GitHub Actions workflow может неожиданно сломаться.

**Как исправлено:**  
В хорошем CI указана фиксированная версия `ubuntu-22.04`.

**Результат:**  
CI стал стабильным и воспроизводимым.

---

### 2. Устаревшая версия `actions/checkout@v1`
**Почему плохо:**  
Старая версия может содержать баги и не поддерживать новые возможности GitHub Actions.

**Как исправлено:**  
Используется последняя стабильная версия `actions/checkout@v4`.

**Результат:**  
Повышена надежность и безопасность workflow.

---

### 3. Установка Node.js через `apt-get`
**Почему плохо:**  
Версия Node.js из apt может быть устаревшей и отличаться от локального окружения.

**Как исправлено:**  
Используется `actions/setup-node@v3` с фиксированной версией Node.js (20).

**Результат:**  
Одинаковая версия Node.js локально и в CI, меньше неожиданных ошибок сборки.

---

## Итог

Исправления сделали CI:
- стабильным
- воспроизводимым
- проще для поддержки
